Android Pizza App

The Android Pizza App is a dynamic and interactive Android application that brings the RU Pizzeria experience to life. Designed for users to seamlessly order their favorite pizzas, the app offers customization options for Chicago and New York-style pizzas, making it both functional and intuitive.

Key Features:
- Pizza Styles: Choose between Chicago-style or New York-style pizzas with distinct crusts and flavors.
- Specialty Pizzas: Preconfigured options such as Deluxe, BBQ Chicken, and Meatzza, each loaded with unique toppings.
- Build Your Own Pizza: Customize pizzas with up to 7 toppings from a selection of 13 available options.
- Dynamic Pricing: Prices adjust dynamically based on pizza size (Small, Medium, Large), with additional costs for extra toppings.
- Order Management: Add multiple pizzas to a single order, view, modify, or delete pizzas in real-time, and automatically calculate subtotals, NJ sales tax (6.625%), and order totals.
- Export Orders: Save order details to a text file for record-keeping or sharing.

Technical Highlights:
- Modern Android UI Components: Uses RecyclerView to dynamically display pizza options and toppings with images, and implements Spinner, AlertDialog, ListView, and Toast for an engaging and responsive user experience.
- Multi-Screen Navigation: Two distinct Android Activities ensure a smooth user flow between pizza customization and order summary.
- Singleton Design Pattern: A centralized data model allows efficient order and pizza data management across the app.
- Custom Resources: Unique launcher icons and pizza images create a polished, professional aesthetic, with `strings.xml` used to eliminate hardcoded text.

Technology Stack:
- Java: Drives core business logic and functionality.
- XML: Powers the intuitive and visually appealing layouts.
- Gradle: Handles build automation and dependency management.
- Android Studio: Provides the development environment for coding, testing, and debugging.

Code Structure:
- app/src/main/java/: Houses the pizza class hierarchy (Deluxe, BBQChicken, Meatzza, BuildYourOwn) and controllers for pizza creation and order management.
- app/src/main/res/: Contains XML layouts for UI screens, launcher icons, and string resources for consistent text management.

The Android Pizza App demonstrates advanced Android development skills, including user-friendly interfaces, responsive design, and efficient data handling. It's an ideal solution for enhancing customer experiences in pizza ordering systems while showcasing modern app development practices.

Key Features:

Backend Setup:

  • Tech Stack: Node.js, Express.js, PostgreSQL

    • POST /auth/login: Log in a user and return JWT.

    • POST /tasks: Create a new task.
    • PUT /tasks/:id: Update an existing task.
    • DELETE /tasks/:id: Delete a task.
  • Runs on: http://localhost:5000

Frontend Setup:

  • Runs on: http://localhost:3000

MySH Project

The MySH Shell Project is a custom-built Unix-like shell terminal developed for the Rutgers University Systems Programming course. This project showcases advanced systems programming skills, including process management, piping, redirection, and command execution. It supports both interactive and batch modes, making it versatile and practical for various use cases.

Key Features:
- Interactive and Batch Modes: The shell operates interactively with a command-line prompt or processes a series of commands in batch mode.
- Command Execution: Executes external programs using execv(), with arguments dynamically managed by a custom-built array list.
- Redirection: Standard output can be redirected to files (existing or new) using dup2(). When combined with pipes, redirection takes precedence.
- Piping: Supports single-pipe operations, directing the output of one program to the input of another using pipe(). Both processes run concurrently.
- Built-In Commands: Includes essential commands such as cd, pwd, which, and exit, offering native shell functionality.
- Wildcard Support: Handles wildcards in command arguments and file paths for flexible command usage.
- Conditional Execution: Implements basic if and else logic, enabling conditional program execution.

Technical Highlights:
- Implements system calls like dup2(), pipe(), fork(), and wait() for process and I/O management.
- Custom arraylist helper functions manage dynamic argument lists.
- Supports commands with up to 3000 characters, 200 tokens, and token lengths of up to 200 characters.
- Provides robust error handling for invalid commands, syntax issues, and file I/O errors.

Usage:
1. Compile the program:
gcc -o mysh mysh.c arraylist.c
2. Run in interactive mode:
./mysh
3. Run in batch mode:
./mysh batchtest.txt

Design Approach:
- Developed in three stages:
1. Built loops for batch and interactive modes and implemented arraylist helper functions.
2. Parsed command strings to determine the executable path, argument list, and input/output files.
3. Wrote execution functions to manage child processes, piping, and redirection using system calls.

Testing:
- Interactive Mode: Tested with various command combinations, redirection, and piping scenarios.
- Batch Mode: Verified functionality with testing files such as batchtest.txt.
- Includes test directories and files such as test, test.txt, and test2.txt to demonstrate edge cases and functionality.

The MySH Shell Project reflects dozens of hours of meticulous development, demonstrating expertise in systems programming and shell design. While minor bugs may exist in rare edge cases, this project offers a functional and robust shell experience with advanced features and attention to detail.

Spell Checker Program

The Spell Checker Program is a project developed for the Rutgers University Systems Programming course. This program efficiently detects and reports spelling errors in one or more text files using a binary search through a dictionary file. Designed for accuracy and flexibility, it supports directory traversal and handles various input scenarios while ensuring precise error reporting.

Key Features:
- Multi-File and Directory Support: Processes individual text files or entire directories, skipping hidden files and directories starting with a ".".
- Binary Search Algorithm: Utilized for fast and accurate dictionary lookups.
- Error Reporting: Clearly identifies spelling errors with their file name, line number, and column position (e.g., file1 (35,8): foom).
- Custom Input Handling: Handles edge cases such as hyphens, capitalization, and trailing punctuation, ensuring robust word processing.

How It Works:
1. Finding Files: The traverseDirectory function opens and identifies all relevant files and directories for processing.
2. Reading and Annotating Words: Extracts words from text files, calculates their positions, and accounts for punctuation and capitalization.
3. Dictionary Lookup: Uses a binary search to check words against the dictionary file, reporting any misspelled words.
4. Main Program: Combines all components in spchk.c, creating a comprehensive spell-checking solution.

Technology Stack:
- C: Core programming language used for efficient file handling and binary search implementation.
- Unix File System: Enables directory traversal and file processing.

Testing:
- Part One: Tests file and directory traversal using part_one_test.c, ensuring all valid files are opened.
- Part Two: Validates word extraction and position annotation with part_two_test.c, handling trailing punctuation and capitalization.
- Part Three: Tests dictionary lookups using part_three_test.c, ensuring accurate word validation through binary search.
- Comprehensive Testing: Combines all parts into spchk.c for end-to-end functionality testing.

Usage:
Compile and run the program using the following commands:
gcc -o spchk spchk.c
./spchk /path/to/dictionary file_or_directory1 file_or_directory2

Examples:
gcc -o spchk spchk.c /usr/share/dict/words testingDirectory test.txt .test.txt
./spchk

The Spell Checker Program is an excellent demonstration of systems programming skills, including efficient data structures, file handling, and robust testing practices. It provides an accurate and user-friendly approach to identifying and addressing spelling errors in text files.
